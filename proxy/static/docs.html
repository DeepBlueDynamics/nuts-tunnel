quickstart

Use the hosted proxy at tunnel.nuts.services. No infrastructure needed.

1. Install the client:

curl -fsSL https://tunnel.nuts.services/install.sh | sh

2. Sign up and get a token from the dashboard.

3. Connect:

nuts-client \
  --proxy wss://tunnel.nuts.services/nuts/ws \
  --token YOUR_TOKEN \
  --service myapp=3000

Your service is live at https://tunnel.nuts.services/myapp/

self-host the proxy

Deploy your own proxy on Google Cloud Run. You'll need gcloud CLI and a GCP project.

1. Clone the repo:

git clone https://github.com/DeepBlueDynamics/nuts-tunnel.git
cd nuts-tunnel

2. Generate a shared secret:

openssl rand -hex 32

3. Build and deploy to Cloud Run:

# copy the Dockerfile to root (Cloud Build needs it there)
cp proxy/Dockerfile .

# build the container image
gcloud builds submit \
  --tag gcr.io/YOUR_PROJECT_ID/nuts-proxy .

# deploy to Cloud Run
gcloud run deploy nuts-proxy \
  --image gcr.io/YOUR_PROJECT_ID/nuts-proxy \
  --region us-central1 \
  --platform managed \
  --allow-unauthenticated \
  --port 8080 \
  --memory 256Mi \
  --min-instances 1 \
  --max-instances 3 \
  --timeout 3600 \
  --set-env-vars "NUTS_TOKEN=YOUR_SECRET_HERE"

4. Map a custom domain (optional):

gcloud run domain-mappings create \
  --service nuts-proxy \
  --domain tunnel.yourdomain.com \
  --region us-central1

Then add the CNAME record shown in the output to your DNS.

client reference

The client is a single static binary. It connects to the proxy over WebSocket and tunnels HTTP requests to your local services.

nuts-client \
  --proxy wss://tunnel.yourdomain.com/nuts/ws \
  --token YOUR_SECRET \
  --service api=8080 \
  --service web=3000 \
  --service docs=4000
flag description env var
--proxy WebSocket URL of the proxy NUTS_PROXY_URL
--token Auth token (shared secret) NUTS_TOKEN
--service name=port mapping (repeatable) -
--config Path to TOML config file -

config file

Instead of CLI flags, you can use a TOML config file:

# nuts-tunnel.toml

proxy_url = "wss://tunnel.yourdomain.com/nuts/ws"
token = "your-secret-token"

[[services]]
subdomain = "api"
port = 8080

[[services]]
subdomain = "web"
port = 3000

[[services]]
subdomain = "docs"
port = 4000
